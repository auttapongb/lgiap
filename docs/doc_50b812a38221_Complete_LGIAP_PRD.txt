1. EXECUTIVE SUMMARY
======================================================================

The LINE Group Intelligence & Archive Platform is an AI-powered system designed to continuously capture, archive, organize, analyze, and retrieve communications occurring within designated LINE groups.

The platform will serve as a permanent institutional memory for communities such as EMBA cohorts, project teams, executive groups, alumni networks, and professional organizations.

The system will preserve conversations, media, links, documents, and interactions while providing advanced AI-powered understanding of complex multi-participant discussions.

The product is not simply a chat recorder. It is a Group Knowledge Operating System that transforms unstructured group conversations into a searchable, organized, AI-assisted knowledge base.

Core capabilities include:

- Continuous monitoring of LINE group chats after bot installation
- Chronological capture of text, stickers, links, images, videos, files, and metadata
- Permanent storage in structured and unstructured repositories
- Daily replay of conversations in original order
- AI-powered summaries, topic mapping, action item detection, and conversation reconstruction
- Dashboard for search, reports, people insights, and knowledge retrieval
- Compliance-aware storage and permission management

Storage Features

3.2 Out of Scope

- Historical messages before the bot is installed in the group
- Private one-to-one LINE chats unless explicitly integrated separately
- Voice call recording
- Video call recording
- Reading deleted content if platform policy disallows it
- Bypassing LINE platform limitations
- Archiving without group consent
- Covert surveillance
- Manual recovery of LINE messages unavailable through the API

3.3 Assumptions

- The bot has been invited to the LINE group.
- LINE Official Account settings allow group participation.
- Webhook is enabled and accessible.
- Group members have been informed that archiving is active.
- The system has permission to store media and metadata.
- Storage provider such as Google Drive, Google Cloud Storage, or AWS S3 is available.
- AI processing is allowed under the data handling policy.

4. FUNCTIONAL REQUIREMENTS
======================================================================

FR-001 GROUP MONITORING

Description:
The bot shall continuously monitor all joined LINE groups after being invited and authorized.

Requirements:
- Monitor 24x7.
- Capture supported LINE webhook events in real time.
- Support multiple groups.
- Support high message volume.
- Detect group ID.
- Detect sender user ID.
- Detect message type.
- Detect timestamp.
- Handle retries and duplicate events.
- Handle webhook failures gracefully.

Acceptance Criteria:
- Messages are captured within 5 seconds of webhook receipt.
- Duplicate messages are not stored twice.
- System records every supported event type.
- System remains operational after restart.

Requirement:
AI shall reconstruct independent conversation threads.

Class scheduling

- Start time
- End time
- Messages included
- Summary
- Decisions
- Action items

Acceptance Criteria:
- AI separates at least 3 simultaneous conversation topics with reasonable accuracy.
- User can click from AI thread view back to original timeline.

Detailed Flow:

1. LINE sends webhook event to webhook gateway.
2. Webhook gateway verifies signature.
3. Raw event is stored immediately.
4. Event is normalized.
5. Message metadata is written to PostgreSQL.
6. Media events are pushed to media download queue.
7. Text content is indexed for search.
8. Embeddings are generated asynchronously.
9. AI pipeline processes summaries, topics, decisions, and action items.
10. Dashboard displays replay, search, reports, and AI insights.

27. RISKS, CONSTRAINTS, AND LIMITATIONS
======================================================================

Platform Limitations:
- LINE may not provide access to historical messages before the bot joins.
- Some media may have time-limited download availability.
- Some user profile data may be unavailable depending on permissions.
- LINE API rate limits may apply.
- Some event types may not be supported.

Privacy Risks:
- Group members may not want permanent archiving.
- Media may include personal information.
- AI summaries may expose sensitive information.
- Participant analytics may feel intrusive.

Technical Risks:
- Webhook downtime can cause missed events.
- Media download failure.
- AI hallucination.
- Topic clustering errors.
- Large storage cost.
- Complex permission model.

Mitigation:
- Obtain group consent.
- Verify LINE webhook signature.
- Store raw events immediately.
- Use queues and retries.
- Add AI source citations.
- Allow admin correction.
- Apply access control.
- Use audit logs.
- Monitor failures.
- Backup regularly.
